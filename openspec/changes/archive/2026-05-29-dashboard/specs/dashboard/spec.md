## ADDED Requirements

### Requirement: Dashboard 按登录角色分流口径

系统 SHALL 提供单一只读看板查询，根据登录主体角色自动决定统计口径：Admin 返回全局口径，Sales 返回个人口径。返回结构对两种角色同构，调用方不需要提供视角参数。看板查询 MUST NOT 修改任何状态，MUST NOT 产生系统日志记录。

#### Scenario: 未登录访问被拒绝
- **WHEN** 未携带有效登录凭证请求看板
- **THEN** 返回 401，不返回任何指标数据

#### Scenario: Admin 与 Sales 均可访问
- **WHEN** 已登录的 Admin 或 Sales 请求看板
- **THEN** 返回 200，且响应包含今日新增数、公海待认领数、本月赢单金额、本月流失率四项指标
- **AND** 不产生任何系统日志记录

### Requirement: 今日新增业务线索数（存量类，按当前归属）

系统 SHALL 统计创建时间落在「今日」区间内的业务线索数量；Admin 计全局，Sales 仅计当前归属为本人的线索。「今日」区间为服务端本地时间，自当日 00:00:00.000 起至次日 00:00:00.000 止（含起不含止）。

#### Scenario: Admin 统计全局今日新增
- **WHEN** Admin 请求看板
- **AND** 今日有 3 条业务线索被创建（无论归属为何，含归属为空者）
- **THEN** 今日新增数为 3

#### Scenario: Sales 仅统计当前归属本人的今日新增
- **WHEN** Sales A 请求看板
- **AND** 今日创建的线索中有 2 条当前归属为 A、1 条归属为他人、1 条归属为空
- **THEN** A 的今日新增数为 2

#### Scenario: 昨日及更早创建的线索不计入今日
- **WHEN** 任一角色请求看板
- **AND** 某线索创建时间为昨日 23:59:59.999
- **THEN** 该线索不计入今日新增数

#### Scenario: 今日零点创建的线索计入今日
- **WHEN** 任一角色请求看板
- **AND** 某线索创建时间恰为今日 00:00:00.000
- **THEN** 该线索计入今日新增数

### Requirement: 当前公海待认领数（存量类，两视角同值）

系统 SHALL 统计当前归属为空且未结束的业务线索数量。该指标对 Admin 和 Sales 均返回全局值，Sales 视角下 MUST NOT 按本人过滤。「未结束」指阶段不为已赢单且不为已流失。

#### Scenario: 统计归属为空且进行中的线索
- **WHEN** 任一角色请求看板
- **AND** 当前存在 5 条归属为空的线索，其中 4 条处于进行中阶段、1 条已流失
- **THEN** 公海待认领数为 4

#### Scenario: 有归属的进行中线索不计入公海
- **WHEN** 任一角色请求看板
- **AND** 某进行中线索当前归属为某 Sales
- **THEN** 该线索不计入公海待认领数

#### Scenario: Sales 视角公海数等于全局值
- **WHEN** Sales A 请求看板，同时 Admin 请求看板
- **THEN** A 看到的公海待认领数与 Admin 看到的相等

### Requirement: 本月赢单总金额（事件类，按事件发生时归属）

系统 SHALL 统计标记赢单事件发生在「本月」区间内的合同金额之和；金额为精确数值，归属取赢单时刻归属快照。Admin 计全局；Sales 仅计赢单时刻归属为本人者。「本月」区间为服务端本地时间，自本月 1 日 00:00:00.000 起至次月 1 日 00:00:00.000 止（含起不含止），以赢单事件时间为锚，而非合同签订业务日期。本月无任何赢单事件时，金额返回 0。

#### Scenario: Admin 统计本月全局赢单金额
- **WHEN** Admin 请求看板
- **AND** 本月发生两次赢单事件，合同金额分别为 100000.00 与 50000.50
- **THEN** 本月赢单总金额为 150000.50

#### Scenario: Sales 仅统计赢单时刻归属本人的金额
- **WHEN** Sales A 请求看板
- **AND** 本月赢单事件中，赢单时刻归属为 A 的合同金额合计 80000.00，归属为他人的合计 60000.00
- **THEN** A 的本月赢单总金额为 80000.00

#### Scenario: 赢单按事件时间而非签订日期归月
- **WHEN** 任一角色请求看板
- **AND** 某合同签订业务日期在上月，但标记赢单事件时间在本月
- **THEN** 该合同金额计入本月赢单总金额

#### Scenario: 无主线索的赢单只计入全局不计入任何 Sales
- **WHEN** Admin 请求看板，且某 Sales A 请求看板
- **AND** 本月某公海线索由 Admin 标记赢单，赢单时刻归属为空
- **THEN** 该笔金额计入 Admin 全局赢单总金额
- **AND** 不计入 A 的赢单总金额

#### Scenario: 本月无赢单时金额为 0
- **WHEN** 任一角色请求看板
- **AND** 本月无任何赢单事件
- **THEN** 本月赢单总金额为 0（数值零，非空值）

#### Scenario: 被停用 Sales 的本月赢单仍计入全局
- **WHEN** Admin 请求看板
- **AND** 某 Sales 本月赢单后被停用
- **THEN** 该笔金额仍计入 Admin 全局赢单总金额

### Requirement: 本月流失率（事件类，按事件发生时归属）

系统 SHALL 计算本月流失率，定义为「本月流失事件数 / 本月结束事件数」，其中本月结束事件数 = 本月赢单事件数 + 本月流失事件数。流失事件时归属取该线索当前归属（流失为终态，归属在流失时刻已冻结），赢单事件时归属取赢单时刻归属快照。Admin 计全局；Sales 仅计事件时归属为本人的赢单与流失事件。「本月」区间定义同本月赢单金额。当本月结束事件数为 0 时，流失率返回空值。

#### Scenario: Admin 统计全局本月流失率
- **WHEN** Admin 请求看板
- **AND** 本月发生 2 次流失事件、3 次赢单事件
- **THEN** 本月结束事件数为 5，流失率为 2/5（40%）

#### Scenario: Sales 按事件时归属统计流失率
- **WHEN** Sales A 请求看板
- **AND** 本月事件时归属为 A 的有 1 次流失、3 次赢单，归属为他人的有若干事件
- **THEN** A 的本月结束事件数为 4，流失率为 1/4（25%）

#### Scenario: 本月结束事件数为 0 时流失率为空值
- **WHEN** 任一角色请求看板
- **AND** 本月无任何赢单或流失事件
- **THEN** 本月流失率返回空值（前端据此渲染为 `--`，而非 `0%`）

#### Scenario: 本月仅有赢单无流失时流失率为 0
- **WHEN** 任一角色请求看板
- **AND** 本月有 2 次赢单事件、0 次流失事件
- **THEN** 本月结束事件数为 2，流失率为 0（0%），非空值

#### Scenario: 无主线索的流失只计入全局不计入任何 Sales
- **WHEN** Admin 请求看板，且某 Sales A 请求看板
- **AND** 本月某归属为空的线索被标记流失
- **THEN** 该流失事件计入 Admin 全局结束事件数与流失分子
- **AND** 不计入 A 的结束事件数与流失分子

#### Scenario: 被停用 Sales 的本月结束事件仍计入全局
- **WHEN** Admin 请求看板
- **AND** 某 Sales 本月有流失或赢单事件后被停用
- **THEN** 这些事件仍计入 Admin 全局结束事件数
